## üìÅ **Asset Types**

### **1. SVG Images**

**Best for:**
- ‚úÖ Icons
- ‚úÖ Logos
- ‚úÖ Vector graphics
- ‚úÖ Scalable UI elements

**Advantages:**
- Scalable without quality loss
- Small file size
- Can change colors dynamically
- Sharp on all screen densities

**Examples:**
```dart

Res.backIcon            // "assets/images/back_icon.svg"
Res.searchIcon          // "assets/images/search_icon.svg"
Res.userIcon            // "assets/images/user_icon.svg"

// From Res class

Res.backIcon            // "assets/images/back_icon.svg"
Res.searchIcon          // "assets/images/search_icon.svg"
```

---

### **2. PNG Images**

**Best for:**
- ‚úÖ Photos
- ‚úÖ Complex graphics
- ‚úÖ Raster images
- ‚úÖ App icons

**Examples:**
```dart
// From Res class
Res.logo                // "assets/images/logo.png"
Res.icon                // "assets/images/icon.png"

// From Res class
Res.logo                // "assets/images/logo.png"
```

**Best for:**
- ‚úÖ Photos with many colors

## üñºÔ∏è **Using Images**

### **Method 1: Using Image.asset (PNG/JPG)**

```dart
// Basic PNG/JPG image
Image.asset(
  Res.logo,
  width: 100,
  height: 100,
  fit: BoxFit.cover,
)

// With error handling

  Res.appIcon,
  width: 80,
  height: 80,

  },

  width: 200,
  height: 200,
  decoration: BoxDecoration(

      image: AssetImage(Res.logo),
      fit: BoxFit.cover,
    ),
    borderRadius: BorderRadius.circular(16),
```

## ‚úÖ **Best Practices**

### **1. Asset Organization**

```dart
// ‚úÖ CORRECT: Use generated asset references
SvgPicture.asset(Res.backIcon)
Image.asset(Res.logo)

// ‚ùå WRONG: Hardcode asset paths
SvgPicture.asset("assets/images/back_icon.svg")
Image.asset("assets/images/logo.png")
```

**Why?**
- Type-safe references
- Auto-complete support
- Compile-time error checking
- Easy refactoring

---

### **2. Choose Right Format**

```dart
// ‚úÖ CORRECT: Use SVG for icons and logos
SvgPicture.asset(
  Res.searchIcon,
  colorFilter: ColorFilter.mode(color, BlendMode.srcIn),
)

// ‚ùå WRONG: Use PNG for simple icons
Image.asset(
  "assets/images/search_icon.png",  // Larger file, not scalable
)

// ‚úÖ CORRECT: Use PNG/JPG for photos
Image.asset(Res.profilePhoto)

// ‚ùå WRONG: Use SVG for complex photos
// SvgPicture.asset(Res.photo)  // SVG not suitable for photos
```

---

### **3. Handle Loading States**

```dart
// ‚úÖ CORRECT: Show loading indicator
CachedImage(
  url: imageUrl,
  width: 100,
  height: 100,
  loadingWidget: CircularProgressIndicator(),
)

// ‚úÖ CORRECT: Handle errors gracefully
Image.asset(
  Res.logo,
  errorBuilder: (context, error, stackTrace) {
    return Icon(Icons.error);
  },
)
```

---

### **4. Optimize Performance**

```dart
// ‚úÖ CORRECT: Specify dimensions
SvgPicture.asset(
  Res.icon,
  width: 24,
  height: 24,
)

// ‚ùå WRONG: Let widget determine size
SvgPicture.asset(Res.icon)  // May cause layout issues

// ‚úÖ CORRECT: Use const for static assets
const Image(
  image: AssetImage(Res.logo),
  width: 100,
  height: 100,
)

// ‚úÖ CORRECT: Cache network images
CachedImage(url: imageUrl)  // Automatically cached

// ‚ùå WRONG: Load network images without caching
Image.network(imageUrl)  // Downloads every time
```

---

### **5. Color Customization**


### **Issue 1: Asset Not Found**

```
Error: Unable to load asset: assets/images/my_icon.svg
```

**Solutions:**

```dart
// 1. Check asset path is correct
SvgPicture.asset(
  "assets/images/my_icon.svg",  // Correct path?
)

// 2. Verify pubspec.yaml includes the asset
flutter:
  assets:
    - assets/images/

// 3. Run flutter clean and rebuild
flutter clean
flutter pub get
flutter run
```

---

### **Issue 2: SVG Not Displaying Correctly**

```
SVG shows blank or has rendering issues
```

**Solutions:**

```dart
// 1. Optimize SVG file
// Remove complex filters, gradients
// Simplify paths

// 2. Specify dimensions explicitly
SvgPicture.asset(
  Res.icon,
  width: 24,   // Specify size
  height: 24,
)

// 3. Check SVG file format
// Ensure it's a valid SVG file
// Remove any unsupported SVG features
```

---

### **Issue 3: Images Taking Too Much Memory**

**Solutions:**

```dart
// 1. Optimize image sizes
// Don't use 4K images for thumbnails

// 2. Use appropriate formats
// SVG for icons
// PNG for transparency
// JPG for photos

// 3. Implement image caching
CachedImage(url: imageUrl)  // Uses memory and disk cache

// 4. Dispose images properly
@override
void dispose() {
  imageCache.clear();
  super.dispose();
}
```

---

### **Issue 4: Font Not Applying**

```
Custom font not showing in app
```

**Solutions:**

```dart
// 1. Check pubspec.yaml configuration
fonts:
  - family: CustomFont
    fonts:
      - asset: assets/fonts/custom_font.ttf

// 2. Use correct font family name
Text(
  'Hello',
  style: TextStyle(fontFamily: 'CustomFont'),  // Match pubspec.yaml
)

// 3. Restart app completely
// Hot reload doesn't always update fonts
flutter run
```

## üéØ **Summary**

### **Key Takeaways**

‚úÖ **Use generated asset references** (Res.dart)
‚úÖ **Choose SVG for icons and logos** (scalable, customizable)
‚úÖ **Use PNG/JPG for photos** (better for complex images)
‚úÖ **Optimize all assets** (reduce file sizes)
‚úÖ **Use CachedImage for network images** (automatic caching)
‚úÖ **Use AppTextStyle for fonts** (automatic font selection)
‚úÖ **Specify dimensions explicitly** (better performance)
‚úÖ **Handle loading and error states** (better UX)

### **Asset Types Quick Reference**

| Type | Use For | Widget | Example |
|------|---------|--------|---------|
| SVG | Icons, logos | `SvgPicture.asset()` | `Res.backIcon` |
| PNG | Photos, app icon | `Image.asset()` | `Res.logo` |
| JPG | Photos | `Image.asset()` | `Res.profilePhoto` |
| Network | User content | `CachedImage()` | Profile images |
| Fonts | Text styling | `AppTextStyle` | All text |

### **Common Widgets**

